# 心理学工具卡 · 写作规范

> 给每一张工具卡一个统一的骨架、专业感和落地感。
> **工具卡 = 给"你用"的，不是给"你看"的。**

---

## 1. 工具卡与 Day 内容的分工

工具卡是 100 天产品里的"工具书层"，与"Day 1-100 每日内容"形成两条平行路线。两者必须风格分明、不互相替代。

| 维度 | Day 1-100 每日内容 | 工具卡 |
|---|---|---|
| 路线 | 文艺路线 · 情绪价值 · 水象 | 专业路线 · 实操工具 · 土象 |
| 主角 | 用户此刻的情绪 | 一个被命名的心理学概念 |
| 进入方式 | 按时间——今天是 Day 38 | 按需要——"我又在还债，去翻 6.3" |
| 寿命 | 当天读完 | 用户毕业 2 年后还能打开 |
| 文体 | 一封写给她的信 | 一本工具书的某一页 |
| 故事 | 核心载体（"她的故事"是主菜） | **不出现**（重复浪费） |
| 心理学小知识 | 给你看 · 散点科普 | 给你用 · 系统工具 |

**判断准则**：把时间感去掉之后还有用，就是工具卡；去掉时间感就没意思了，就该写到 Day 里。

---

## 2. 标准头部模板

每张工具卡都用以下结构开头，不可省略：

```markdown
---
title: [编号] [工具卡名]——[一句话核心]
type: 工具卡
created: YYYY-MM-DD
tags: [#工具箱, #所属维度, #核心概念, #100天]
related: [[wiki/共享知识/心理学工具箱/index.md]], [[相关其他工具卡]]
---

# [编号] · [工具卡名]

> **[一句话精华——给用户记住的那句金句]**

---

🧰 **心理学工具 · 当下即用**

| 项目 | 内容 |
|---|---|
| 理论依据 | [核心理论]（[作者], [年份]）+ [辅助理论]（[作者], [年份]） |
| 流派 | [一级流派] · [二级流派] |
| 适用 | [场景 1] · [场景 2] · [场景 3] |
| 周期 | [训练周期，如 21 天起步 / 单次干预 / 长期练习] |

---
```

**头部 4 行字承载的信息密度——这是土象专业感的第一锤子**：
- "理论依据" 给可信度
- "流派" 给体系定位
- "适用" 帮用户判断"这张卡是不是给我的"
- "周期" 给用户合理预期，避免"练 3 天没效果就放弃"

---

## 3. 六维度流派挂靠地图

每张工具卡必须挂靠至少一个真实存在的心理学流派 + 至少一位代表性研究者/治疗师。这是"专业感"的基本盘。

| 维度 | 主挂流派 | 主要理论锚点 | 代表人物 |
|---|---|---|---|
| **1 自我价值与配得感** | 自体心理学 + 客体关系 | 镜像自体客体 / 真实自体 vs 虚假自体 / 萨提亚冰山 | Heinz Kohut · Donald Winnicott · Virginia Satir |
| **2 关系边界与自我主权** | 行为治疗 + 个体心理学 + 家庭系统 | 系统脱敏 / 课题分离 / 自我分化 / NVC | Joseph Wolpe · Alfred Adler · Murray Bowen · Marshall Rosenberg |
| **3 决策自主与信任** | 接纳承诺疗法 ACT + 积极心理学 | 价值澄清 / 认知解离 / 心理灵活性 / 心流 | Steven Hayes · Mihaly Csikszentmihalyi |
| **4 情绪自主与稳定** | 多层迷走神经 + 创伤治疗 + 内在家庭系统 | 神经觉 / 安全锚点 / 躯体体验 / Parts Work | Stephen Porges · Bessel van der Kolk · Peter Levine · Richard Schwartz |
| **5 行动通道与能量流动** | 完形疗法 + 行为激活 + WOOP | 此时此地 / 未完成事件 / 心理对照 | Fritz Perls · Gabriele Oettingen |
| **6 财富容器与丰盛感** | 神经心理学 + 家族系统排列 + 代际创伤 | RAS / 家族脚本 / 代际传递 | Stephen Porges · Bert Hellinger · Mark Wolynn |

**使用方法**：
- 写新卡时，先在维度对应行里挑 1 个主理论 + 1 个辅助理论，写进头部"理论依据"
- 一个流派可以跨维度复用（如 IFS 在 4 和 5 都出现）——这是体系内部的勾连，不是抄袭
- 若你想引入此表外的流派，先增补到此地图再写卡

## 4. 标准段落结构

头部之后，按以下五段顺序写。**不再使用旧版"🌰 一个生活例子"段——故事归 Day 内容**。

```markdown
## 🪞 这张卡解决什么问题

[3-5 个反问句，命中目标用户的具体痛点]
[1 段定性总结——这不是 X，是 Y]

## 🧠 核心原理

[150-300 字。专业术语必须出现，但用白话注释。
讲清楚机制：为什么会这样？身体/认知/神经层面发生了什么？
关键概念给出英文术语 + 提出者。
不抒情、不煽情、不比喻过载——保留 1 个核心比喻作为锚点即可。]

## 🛠️ 怎么用

### 第一步：[识别 / 觉察类动作]
[用表格、清单、对照——可视化的工具。少用整段文字。]

### 第二步：[核心训练动作]
[梯度表 / 句式表 / 步骤表 二选一。土象产品的核心是"看得见摸得着"。]

### 第三步：[整合 / 巩固动作]
[给周期、给频次、给身体反馈节点。如"第 7 天 / 第 14 天 / 第 21 天身体会发生什么"。]

## ⚠️ 重要提醒

[针对用户最容易陷入的误区——通常是把工具用反、用过头、或用偏。
1 段反向澄清。这是工具卡里**唯一允许保留情感语言的段落**——因为它要把人从误区里拽回来。]

## 💎 一句话总结

**[3 行内的金句。这是用户合上手机后会带走的那句话。]**
```

**砍掉的段落**：
- ❌ 🌰 一个生活例子（归 Day 内容的"她的故事"段）
- ❌ 📌 在 100 天里出现在哪几天（破坏复用性，且这是产品后端逻辑，不该暴露给用户）

## 5. 文字风格准则：水象 → 土象的转换

工具卡的文字必须**比 Day 内容硬 30%**。具体怎么硬：

### 5.1 加 · 三件事

| 加什么 | 例子 |
|---|---|
| **专业术语** | "应激反应" / "习得性无助" / "防御性副交感反应" / "认知解离" / "代际传递" |
| **作者 + 年份** | "（Wolpe, 1958）" "（Porges, 1994）" "（Bowlby, 1969）" |
| **机制描述** | "下丘脑-垂体-肾上腺轴" / "前额叶皮质" / "杏仁核" / "镜像神经元" 等真实生理结构名称 |

### 5.2 减 · 三件事

| 减什么 | 反例（Day 风格） | 正例（工具卡风格） |
|---|---|---|
| **诗化比喻过载** | "你身体里管拒绝的那块肌肉，从你还小的时候就被废用了" | "这是早期依恋系统在反复'说不即失关系'的反馈下形成的固化神经回路——'不'这个动作的反应阈值持续偏高。" |
| **第二人称煽情** | "你不需要变成另一个人，你只需要在颤抖里说出口那一次" | "训练目的不是消除焦虑，而是建立'焦虑共存下仍可执行'的行为模式。" |
| **散文式留白** | 多个独立短句换行，靠节奏感打人 | 段落整合，靠信息密度打人 |

### 5.3 不动 · 三件事

| 保留 | 原因 |
|---|---|
| **第二人称"你"** | 这是产品语气，不是论文。不要切第三人称。 |
| **核心比喻 1 个** | 给工具一个能记住的画面。但只能 1 个，不能 3 个。 |
| **白话化解释** | 专业术语后面必须跟一句白话注释，不允许"裸术语"。 |

### 5.4 一个具体重写示范

```
旧版（情感路线）：
"不是你性格软。是你身体里管'拒绝'的那块肌肉——从你还小的时候就被废用了。"

新版（专业路线）：
"讨好型反应不是性格特征，而是早期依恋关系（Bowlby, 1969）反复反馈
'说不 = 失去关系'后形成的固化神经回路。在这套回路里，'拒绝'这个
动作长期未获得正向反馈，因此响应阈值持续偏高——表现为想拒绝时心
跳加速、喉咙发紧、说出口后强烈反悔冲动。"
```

后者**没有变难懂**——只是把"肌肉被废用了"这个比喻换成了"神经回路 + 响应阈值"这个机制描述，并给出依恋理论的来源。专业感和落地感同时在。

## 6. 命名规则

### 文件命名

```
[维度编号].[卡序号]-[工具名].md

例：2.2-拒绝肌肉训练.md
    6.1-RAS 网状激活系统.md
```

工具名可以保留比喻（如"拒绝肌肉训练"），但**头部"理论依据"必须挂靠真实流派**——这样既好记又专业。

### Frontmatter title

```yaml
title: [编号] [工具卡名]——[一句话定位]
```

例：
- `title: 2.2 拒绝肌肉训练——20 年没练的肌肉，第一次拉伸当然疼`
- `title: 6.1 RAS 网状激活系统——你的大脑只看见你"相信"的东西`

## 7. 维护要求

每写完一张工具卡，**必须同步更新**以下两个 index：

1. **维度子文件夹的 index.md**（如 `2-关系边界与自我主权/index.md`）
   - 工具卡列表表格里追加这张卡
   - 若这张卡是这个维度第一张完成的，需新建子 index

2. **工具箱总 index.md**（`心理学工具箱/index.md`）
   - 对应维度表格里更新这张卡的链接
   - 进度区把这个维度的状态从"待写"改成已完成的张数（如"4 张已完成 / 共 4 张"）

写完后还需在**根目录 log.md** 追加一行 ingest 记录。

---

## 8. 验收清单

写完一张卡后自检：

- [ ] 头部 frontmatter 完整（title / type / created / tags / related）
- [ ] 头部信息条 4 行齐全（理论依据 / 流派 / 适用 / 周期）
- [ ] 理论依据挂靠的流派和人物**真实存在**（不可生造）
- [ ] 五段结构齐全（解决什么问题 / 核心原理 / 怎么用 / 重要提醒 / 一句话总结）
- [ ] 没有"🌰 一个生活例子"段
- [ ] 没有"📌 在 100 天里出现在哪几天"段
- [ ] 核心原理段至少出现 2 个专业术语 + 1 个作者引用
- [ ] "怎么用"段至少含 1 个表格（梯度表 / 句式表 / 对照表）
- [ ] 子 index 和总 index 已同步更新
- [ ] log.md 已追加记录
