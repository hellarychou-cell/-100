这份文件给以后改小字用。原则是：能放在根目录 Markdown 的正文，就不要去 React 里改；按钮、导航、入口提示这类界面小字，才改 `src/app` 或 `src/components`。

| 页面 | 线上路径 | 文案来源 |
|---|---|---|
| Day 1-7 故事页 | `/day/1` 到 `/day/7` | `成她-Day1-7.md` |
| 理念页 | `/philosophy` | `成她-理念页.md` |

## 改完怎么上线

1. 在 `/Volumes/PS2000/成她` 里改对应文件。
2. 跑基础检查：

```bash
npm test
npm run lint
npm run build
```

3. 同步到 GitHub/Vercel：

```bash
/Volumes/PS2000/成她/scripts/sync-to-github.sh "你的更新说明"
```

4. 如果也要同步腾讯云，继续按 `项目继续修改与同步指南.md` 的腾讯云步骤执行，并记得排除 `.env.production`。

## 这次文案刷新规则

- `成她-Day1-7.md` 是 Day1-7 的权威来源。
- `成她-理念页.md` 是理念页正文的权威来源。
