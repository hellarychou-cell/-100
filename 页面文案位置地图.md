# 页面文案位置地图

这份文件给以后改小字用。原则是：能放在根目录 Markdown 的正文，就不要去 React 里改；按钮、导航、入口提示这类界面小字，才改 `src/app` 或 `src/components`。

| 页面 | 线上路径 | 文案来源 |
|---|---|---|
| Day 1-7 故事页 | `/day/1` 到 `/day/7` | `成她-Day1-7.md` |
| Day 1-7 觉醒剧场 | `/day/1` 到 `/day/7` | 剧场正文和选项来自 `成她-Day1-7.md`，AI 接话锚点来自 `src/lib/day-ai-anchors.ts` |

| 身体驿站 | `/body-station`、`/body-station/[day]` | `成她-身体驿站页.md` |

| 工具卡/神秘卡工具内容 | `/collection` | `心理学工具箱/` |

## 改完怎么上线

1. 在 `/Volumes/PS2000/成她` 里改对应文件。
2. 跑基础检查：

```bash
npm test
npm run lint
npm run build
```

3. 同步到 GitHub/Vercel：

```bash
/Volumes/PS2000/成她/scripts/sync-to-github.sh "你的更新说明"
```

4. 如果也要同步腾讯云，继续按 `项目继续修改与同步指南.md` 的腾讯云步骤执行，并记得排除 `.env.production`。

## 这次文案刷新规则

- `成她-Day1-7.md` 是 Day1-7 的权威来源。
- Day1-7 的觉醒剧场正文、A/B/C 或 X/Y 选择，也优先改 `成她-Day1-7.md`。
- 如果只是改 AI 后台接话锚点的小句子，改 `src/lib/day-ai-anchors.ts`。
- `成她-理念页.md` 是理念页正文的权威来源。
- `src/lib/content.ts` 只做旧数据兜底，不再作为 Day1-7 正文主来源。
- 修改 Markdown 后，需要重新构建/同步，线上才会看到变化。
