## 当前代码实现交接（2026-06-07）

### 当前工程位置
- 本地项目：`/Volumes/PS2000/成她`
- 技术栈：Next.js App Router + TypeScript + Tailwind + Recharts + Supabase JS
- 当前不是 git 仓库，接手时请先用文件清单和 `npm run lint` / `npm run build` 判断状态。

### 已补上的产品功能
- 登录 / 注册：
  - 页面：`src/app/auth/page.tsx`
  - 组件：`src/components/AuthForm.tsx`
  - Supabase 配置存在时走 `supabase.auth.signUp` / `signInWithPassword`
  - 未配置 Supabase 时走本地演示账号
  - 本地演示规则：手机号以 `9999` 结尾视为已开通会员；其他手机号为待开通
- 会员权限拦截：
  - 组件：`src/components/AuthGate.tsx`
  - 已接入：`/home`、`/day/[day]`、`/treasure`、`/cards`、`/ai-summary`
  - `/knowledge` 保持公开说明页；真正 Day 正文需要会员
- 完整 42 题测评：
  - 数据：`src/lib/assessment.ts`
  - 组件：`src/components/AssessmentFlow.tsx`
  - 页面：`src/app/assessment/page.tsx`
  - 规则：6 维度 × 7 题，答完整后才生成报告
  - 提交后保存：localStorage；若已登录 Supabase，则写入 `assessments`，并 upsert `progress.current_day`
- 测评前资料：
  - 组件：`src/components/AssessmentProfileForm.tsx`
  - 页面：`src/app/assessment/profile/page.tsx`
  - 保存：localStorage；若已登录 Supabase，则 upsert `profiles`
- 报告生成：
  - 组件：`src/components/AssessmentResultClient.tsx`
  - 页面：`src/app/assessment/result/page.tsx`
  - 优先读取 Supabase 最新测评；没有则读 localStorage；都没有则提示先测评
- 理念页 / 知识库页内容接入：
  - Markdown 读取：`src/lib/markdown.ts`
  - 理念源文档：`成她-理念页.md`
  - 知识库源文档：`成她-知识库页.md`
  - 页面：`src/app/philosophy/page.tsx`、`src/app/knowledge/page.tsx`

### 另一个 AI 接手时先做什么
1. 先读这一段交接，再读：
   - `src/lib/assessment.ts`
   - `src/components/AuthForm.tsx`
   - `src/components/AuthGate.tsx`
   - `src/components/AssessmentFlow.tsx`
   - `src/components/AssessmentResultClient.tsx`
2. 跑：
   - `npm run lint`
   - `npm test`
   - `npm run build`
3. 如果要继续补真实产品能力，优先级是：
   - Supabase RLS policies 和后台加 30 天的真实写入权限
   - `/admin` 用户列表接 Supabase `profiles` + `memberships`
   - `/home` 读取真实 `progress`
   - `/cards`、`/ai-summary` 读取真实用户数据
   - AI 对话真实接模型与写入 `ai_conversations`

- 生产上线前必须在 Supabase 开启和校验 Auth 配置、表权限、RLS 策略。
