## 🎬 觉醒剧场 · 写作规范

### 格式骨架（约 1200-1500 字目标 · 2026-06-28 双选择点升级版）

```
1. 开篇共享段（200-250 字）
   - 第二人称：「你叫 XX。今年 X 岁。」
   - 时间场景具体到时刻（「周日下午 3 点」而非「某天」）
   - 建立核心张力（一个动作前的「那一秒」）
   - 自然带出"某人问你一句话 → 8 双眼睛转向你 → 你开了口……"

2. 第一选择点（A/B/C · 每项一行内心独白）· 外界压力下的反应
   - A = 最熟悉的自动反应（讨好/逃避）
   - B = 中间档（说出口但收回去）
   - C = 突破档（不直接对抗，但做一件保护自己的小事）

3. 三条分支（每条 180-220 字 · 各自完整起承转合）
   - 各自有对方反应金句
   - 各自有「那一秒」的细节
   - 不在分支里做底层代码揭示
   - 三条结尾都收在"她自己的那一刻"——不是别人的评价

4. 共同观点段·上半（100-150 字 · 沉淀今年的让渡 · "共同观点段"5 字不在页面展现，三条分支完成后直接衔接）
   - 用"夜里你躺下"或类似的场景把三条分支收回到同一个"你"
   - 列举"这一年里"反复让出去的具体场景（年初/春天/夏天……）
   - 严守纪律 1：不溯源童年，只列今年/最近的让渡
   - 不牵 AI、不出指令、不教方法
   - 结尾自然引到"这一刻你心里冒出来的是——"

5. 第二选择点（X/Y · 内在独白二选一）· 夜里一个人时的内在动作 🆕
   - X = 延续旧程序（"算了，明天又是新的一天"型）
   - Y = 松动旧程序（"我累了。从今晚开始，我想试着不再这么累"型）
   - 不评判好坏——选 X 也是真实，选 Y 也是真实
   - 用「**X · "……"**」「**Y · "……"**」格式
   - 每项是一句完整的内在独白

6. 两条分支（每条 100-150 字 · 各自走完一段心路）🆕
   - X 分支：用户继续延续旧程序后的细节（关灯/翻身/明天又一天）
   - Y 分支：用户开始松动旧程序后的细节（坐起来/打开备忘录/写下一句话）
   - 两条结尾都收在"今晚的你"——不强行升华
   - 不在分支里做底层代码揭示

7. ### 💡 底层代码揭示（30-50 字 · 阅读高潮点睛）
   - 用 `### 💡 底层代码揭示` 独立小标题
   - 一句话点出"你不是 X，是 Y"
   - 无论第二选择选 X 还是 Y，都能稳稳收回到同一句话
   - 与后续金句卡互相呼应（金句卡是 AI 后的留念，揭示是故事尾的收束）

8. 幕间卡承接 AI（≤ 50 字 · 给开发的过渡 UI）
   - 用 `> *↓ 幕间卡（给开发）：……→ [开始对话]*` 注明
   - 文案：把今天故事的核心问题反问回用户
   - 例：「今晚有人问你"你自己想要什么"。你愣了 3 秒。如果下次再有人问你这个问题，你会怎么回答？」
```

**字数总账**：开篇 200-250 + 3 条分支 540-660 + 共同观点段·上半 100-150 + 第二选择点+ 2 条分支 200-300 + 💡 揭示 30-50 + 幕间卡 ≤50 = **约 1200-1500 字**

**两次选择的设计意图**：
- 第一选择 = 「**面对外界那一秒，你怎么反应**」（讨好 / 收回 / 保护自己）
- 第二选择 = 「**夜里一个人时，你怎么对自己**」（继续否认 / 开始看见）
- 第二选择**比第一选择更内在**——一个人独处时的真实反应才是觉醒的真正起点

### 写作规范（与原「她的故事」一脉相承）

- 所有人物**全部虚构**，原型来自个案 + 生活观察
- 用户群 25-45 岁，名字/城市/行业随机化
- 反常识瞬间：「你以为她会问赚钱吗，结果她说了 5 个字」
- 细节丰富：灰色羊绒衫 / 那筷子虾 / 旧搪瓷碗 / 11:30 的厨房
- 💡 底层代码反转（「你不是 X，是 Y」）**放在故事尾的独立段**（`### 💡 底层代码揭示`），与金句卡互相呼应：
  - 故事尾💡 揭示 = 阅读高潮点睛（用户读完三条分支 + 共同观点段后，需要一句话总结今天看见的是什么）
  - 金句卡 = AI 对话完成后的留念（同一概念的再次显现，用户已经被 AI 带进去过，金句卡是把"今天我看见了"封存进档案）
  - 两处文案**不完全重复**——故事尾💡 是叙事语气（"你不是…是…"），金句卡是诗化语气（带换行/留白/署名）
  - 👉 示例对照见 Day 1：故事尾💡「你不是不会选。是你把'自己要什么'的权利一次次让了出去。让得太顺手，让到今晚生日饭桌上。」vs 金句卡「你不是不会选 / 是你把"自己要什么"的权利 / 一次次让了出去 / —— Day 1」

### 🚫 两条硬纪律（必读 · 来自小红书引流文案 v3 + 身体驿站页规范）

**纪律 1 · 不溯源童年（叙事主轴是「你今年让出去的权利」）**

❌ 禁止：「3 岁那年妈妈说……」「8 岁那年家里总说……」「小学三年级你妈带你买衣服……」
✅ 替代：把 💡 揭示放在「**这一年 / 这一个月 / 最近**」级别的具体事件上——「年初你想报写作班，没报；春天你想剪短发，没剪……」
✅ 例外：如果非要用童年痕迹，**最多一句、且立刻收回到「但你今天可以选择不再让」**

**原因**：v6 主轴已经从「童年被装的程序」换到「你今年/此刻让出去的权利」（人生主动权）。溯源童年违反 IP 主轴换向，也容易掉进受害者剧本。

**纪律 2 · 破折号克制（每段不超过 1-2 个）**

❌ 禁止：每段 3 个以上「——」，连续短句靠破折号串
✅ 替代：用句号、逗号、换行、独立短句替代多余破折号
✅ 保留：节奏关键点的破折号（如「8 双眼睛都转向你——你开了口」），但每段最多 1-2 处

**原因**：破折号过密会让文字变成"散文体训练营文案"，丢失角色独白的真实节奏。

### 3 档分支模式（v6.2 互动节奏控制 · 2026-06-28）

> 说明：v6.2 起觉醒剧场分 3 档分支模式。每天具体走哪档，已在 §十二 100 天排期表「分支模式」列锁定。
> 旧版「3-2-1 完整版 vs 简版」分级废弃 → 改为本节 3 档结构。

#### 2 分支 · 完整双选择点版（47 天 · 47%）

完整 8 段觉醒剧场：开篇 → A/B/C 第一选择 → 3 条分支 → 共同观点段·上半 → X/Y 第二选择 → 2 条分支 → 💡 揭示 → 幕间卡。字数 1200-1500。

**写法**：见 §六 觉醒剧场写作规范 · 完整 8 段格式骨架 · 范本 Day 1 / Day 2。

**适用**：W1 留存关键 + 重建期/创造期主体 + 大部分卡点深挖日。

#### 1 分支 · 单选择点版（28 天 · 28%）

简化 6 段觉醒剧场：开篇 → A/B/C 第一选择 → 3 条分支 → 共同观点段（一次走完不分上下半）→ 💡 揭示 → 幕间卡。字数 900-1100。

**写法**：
- 保留第一次选择（A/B/C）
- 删掉第二次选择（X/Y）+ 两条 X/Y 分支
- 共同观点段一次走完（不再"上半 → 第二选择 → 下半"）
- 💡 揭示和幕间卡照常

**适用**：觉醒期/理解期/创造期的中间档；用户能量没满到需要 2 次选择的日子。

#### 0 分支 · 纯叙事版（25 天 · 25%）—— 分 3 子型

##### 0α 单线流（11 天 · 第二人称沉浸 · 历史叙事用）

第二人称单线故事：开篇 → 沉浸式走完整个故事（无选择）→ 共同观点段 → 💡 揭示 → 幕间卡。字数 1000-1200。

**和旧版"她的故事"的关键区别**：
- 人称用「你叫 XX」（不是「她叫 XX」）—— 沉浸感升级
- 共同观点段 + 💡 揭示 + 幕间卡完整保留 —— 叙事高潮不丢
- 只是去掉 A/B/C 选择 —— 故事线本身已经足够有力，无需选择推进

# 八、AI 对话设计标准（关键 ⭐）

## 设计原则

1. 每天**只问 1 个清晰的问题**（不再并列多个钩子让用户选择困难）
2. 上方 1 行小字标注**今日方法**（如"第四象限·苏格拉底"）
3. 折叠区放**适用场景说明**——一行小字，不展开
4. 正文部分**像和真人对话**——保留"跟我说说"、"我会陪你看见"
5. 提供**对应的 IFS-AI 手册指令链接**——用户拿到产品就有完整提问工具

```

[一行小字适用场景说明]
完整指令在【IFS-AI 提问手册·XXX】。
</details>

> **跟我说说——**
>
> [今天主题相关的具体单问题]
>
> 我会陪你看见——
```

## AI 接话锚点（v6.2 对接觉醒剧场 · 2026-06-28）

### 设计原理

用户在觉醒剧场做选择（A/B/C 或 X/Y）→ 前端记录选择 → AI 对话开场第一句**直接接住用户在剧场里的状态**——不复述剧情，直接问出她没说出口的问题。

### 技术实现

每天定义 3-5 个"接话锚点"（用户选 A/B/C/X/Y 后分别处于什么状态），前端传给后端，后端拼进 system prompt：

```
用户刚刚在觉醒剧场里的状态：【锚点内容】。
你的第一句话要直接接住她此刻的状态——不要复述剧情，直接问出她没说出口的问题。
```

AI 根据锚点自己生成开场句，不写死句子，语气更自然。

# 十七、每日页面 UX 过渡设计（给开发看）

> 2026-06-27 新增。
> 目标：用户被"牵着走"，自然看完整天内容，15 分钟不中断、不流失。

## 核心设计原则

- **不靠弹窗驱动流程**（弹窗 = 广告感 = 破坏沉浸）
- **靠页面设计本身的"牵引感"**（Peek + 渐显引导条）
- **唯一例外**：觉醒剧场 → AI 对话的节点用「幕间卡」（模式切换需要仪式感）

## 各节点过渡方式

### 节点 1：觉醒剧场 → AI 对话（唯一特殊节点）

**用「幕间卡」——不是弹窗**

原因：从「读故事」到「跟 AI 说心里话」是全天**唯一的模式切换**（被动接收 → 主动开口），需要一个心理停顿，否则用户在「消费模式」里滑过 AI 输入框。

UI 实现（2026-06-27 Day 1 实战样本）：

```
[觉醒剧场最后一行：💡 底层代码揭示 段]
你不是不会选。
是你把"自己要什么"的权利，
一次次让了出去。

────────────────────────────
│                          │
│  今晚有人问你             │
│  "你自己想要什么"。       │
│  你愣了 3 秒。            │
│                          │
│  如果下次再有人问你       │
│  这个问题——              │
│  你会怎么回答？           │
│                          │
│     [ 开始对话  →  ]    │
│                          │
────────────────────────────
```

- 全屏宽度卡片，背景用品牌暖色
- 不遮挡上方内容（不是 modal）
- 按钮点击才进入 AI 输入框
- 停留感：给用户 0.5 秒的「我准备好了」
- **幕间卡是「承接桥」不是「结案」**——文案必须**反问回用户**，把今天故事的核心问题抛给她，而不是替她总结。她带着这个问题进 AI 输入框，才是顺势开口。

---

### 节点 2：AI 对话 → 身体小语

**用「渐显引导条」**

AI 对话完成后，页面底部出现一条窄条（约 48px 高），文案随今日主题变化：

```
[AI 对话完成 → 底部渐显]
───────────────────────────────────
让今晚说过的话，在身体里再松一松  →
───────────────────────────────────
```

- 不打断 AI 对话区域
- 用户可以忽略继续停在 AI，也可以点击进入
- 文案不用「下一步」「继续」等 App 词

---

### 节点 3：身体小语 → 神秘卡

**用「Peek 效果」**

用户滑到身体小语底部时，神秘卡的顶部已经**半露出约 40px**，牌面微微模糊——天然勾起「那张卡是什么」的好奇心。

```
[身体小语底部]
... 就让那个紧着的地方——
   慢慢松开。

────────── ↓ 已可见（模糊）──────────
✦ 今日抽卡 ✦                [翻牌按钮]
    [卡片背面隐约可见]
```

---

### 节点 4：神秘卡 → 轮换内容（当天有则出现）

**用「Peek 效果」**

同上逻辑，轮换内容的标题栏半露出：

```
[神秘卡区域下方]
... 今天还有一段话想跟你说。

────────── ↓ 已可见（微模糊）──────────
❹  心理学小知识  ──
   "内摄"——你以为的"你"……
```

---

### 节点 5：轮换内容 → 整天散场尾韵

**无引导，静态展示**

轮换内容结束后，散场尾韵**直接出现在页面末尾**，不加任何按钮或引导：

```
[轮换内容结束]

────────────────────────────
（散场尾韵 · 静态文字 · 无按钮）

[整天散场尾韵文案]

────────────────────────────
← Day N-1            Day N+1 →
```

## 过渡方式汇总表

| 节点 | 方式 | 是否可忽略 | 触发时机 |
|---|---|---|---|
| 觉醒剧场 → AI 对话 | **幕间卡（嵌入式）** | 否（需主动点击）| 故事结尾显示 |
| AI 对话 → 身体小语 | **渐显引导条** | 是 | AI 完成后底部出现 |
| 身体小语 → 神秘卡 | **Peek 效果** | 是（自然下滑）| 始终半露出 |
| 神秘卡 → 轮换内容 | **Peek 效果** | 是 | 始终半露出 |
| 轮换内容 → 散场尾韵 | **静态展示** | — | 页面末尾 |

## 母题点睛位置（四层 · 2026-06-27 Day 1 实战修正）

| 层 | 位置 | 说什么 | 形式 |
|---|---|---|---|
| ① | 觉醒剧场共同观点段 | 列举今年/最近的具体让渡（不点睛、只沉淀）| 故事语气，不教学 |
| ② | 觉醒剧场 💡 底层代码揭示 | 「你不是 X，是 Y」叙事点睛 | 独立段 + 小标题 |
| ③ | AI 对话过程中 | 让她自己说出来（教练手法）| AI 追问 |
| ④ | 💎 金句卡 | 同一概念的诗化封存 | 金句格式 + 出处 |

❌ 散场尾韵里不做母题点睛——散场只留"一个具体物"（一只碗、一盏灯、一句话）

**四层之间的关系**：
- ① → ② 是叙事的高潮收束（用户读完故事自然到达揭示）
- ② → ③ 中间用幕间卡承接（不再次说揭示，反而把核心问题反问回用户）
- ③ → ④ AI 完成后金句卡解锁，是"我看见了"的留念证物
- 四层文案**不完全重复**——同一概念在不同位置用不同语气出现，形成回响而非重复
